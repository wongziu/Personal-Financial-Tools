# 个人投资信息与交易决策系统：完整落地规范 V1.0

## 0. 核心结论

你需要建立的不是“持仓表 + 盈亏表”，而是一个可以回答以下问题的系统：

1. **我现在拥有什么，真实净值是多少？**
   2([投资者网站][1])*下一笔交易是否符合规则，而不是符合当下情绪？**

该系统由 **13 张表、4 类规则、5 个固定流程、4 套复盘模板** 构成。

> **关键设计原则：**
>
> * 事实账本与观点研究必须分开。
> * 长期配置、主动投资、短期交易、实验策略必须分开。
> * 收益评价与风险控制必须分开。
> * 买入理由与卖出理由必须在交易前形成，而不是事后解释。
> * 组合表现必须排除入金、出金等外部现金流影响后再评价策略能力。

资产配置、分散化与再平衡的目的，是控制组合风险水平而非预测市场；集中于某个标的、行业或市场可能放大损失；费用会长期侵蚀回报；保证金交易可能造成超过初始投入的亏损；时间加权收益能够降低外部现金流对策略评价的干扰。([投资者网站][1])1 四层结构

| 层级     | 目标                  | 对应表格    |
| ------ | ------------------- | ------- |
| 事实层    | 准确记录账户、交易、现金流、持仓与费用 | S01–S05 |
| 研究层    | 记录信息来源、投资逻辑、反方证据与事件 | S06–S08 |
| 决策与风险层 | 控制仓位、风险、交易与例外行为     | S09–S11 |
| 复盘与治理层 | 评价结果、规则、错误与系统有效性    | S12–S13 |

## 1.2 数据流

```text
账户 / 交易 / 现金流 / 行情
          ↓
持仓快照与组合净值
          ↓
风险仪表盘与限制检查
          ↓
投资论点 + 信息证据 + 交易决策单
          ↓
执行交易
          ↓
月度复盘 / 季度复盘 / 规则更新
```

# 3. 全系统统一填写标准

这一部分不是单独的一张表，但决定系统能否长期使用。

## 3.1 数据填写总标准

| 项目   | 强制标准                                           |
| ---- | ---------------------------------------------- |
| 日期格式 | 统一使用 `YYYY-MM-DD`；涉及交易时间时使用 `YYYY-MM-DD HH:MM` |
| 基准货币 | 只允许一个基准货币，例如人民币 `CNY`                          |
| 原币记录 | 海外资产同时记录原币金额与换算后基准货币金额                         |
| 数据来源 | 所有数字必须标记来源：券商、基金公告、上市公司公告、交易所、人工估算             |
| 空值规则 | 不知道填写 `Unknown`；不适用填写 `N/A`；禁止空白               |
| 修改规则 | 原始交易记录不得覆盖；错误通过新增“更正记录”修正                      |
| 标识符  | 所有账户、标的、策略、论点、交易均建立唯一 ID                       |
| 数字精度 | 价格按交易所或券商精度；金额保留两位；比例保留两位百分数                   |
| 费用记录 | 佣金、平台费、印花税、交易税、融资利息、基金费率必须分别记录                 |
| 文本标准 | 不允许只写“看好”“低估”“趋势好”；必须写可验证依据                    |
| 引用标准 | 影响买卖决策的重要事实必须附原始来源和发布日期                        |
| 版本标准 | 研究观点更新时保留旧版本，不得删除原判断                           |

## 3.2 信息证据等级

| 等级 | 类型                          |  是否可直接支撑交易 |
| -- | --------------------------- | ---------: |
| A  | 财报、公告、招股书、监管文件、基金定期报告、券商对账单 |          是 |
| B  | 公司业绩会原文、交易所问答、行业协会正式数据      |  可以，但需交叉验证 |
| C  | 卖方报告、基金经理公开观点、权威媒体采访        |       只能辅助 |
| D  | 自媒体、论坛、群聊、短视频、匿名爆料          | 不可直接作为买卖依据 |

### 填写规则

* **事实字段**只能来自 A 或 B 级来源。
* **观点字段**可以引用 C 或 D 级来源，但必须标记为观点。
* 任何来自 D 级来源的信息，若触发交易，必须在交易前找到更高等级证据。

# 4. S00：投资政策与系统参数表

这是整个系统的最高层规则。每半年复核一次，发生重大收入、家庭责任、负债或资金用途变化时立即复核。

## 4.1 字段设计

| 字段         | 类型    | 必填 | 填写标准                             |
| ---------- | ----- | -: | -------------------------------- |
| 系统版本号      | 文本    |  是 | 如 `IPS-2026-H1-V1`               |
| 基准货币       | 枚举    |  是 | CNY / USD / HKD 等，只能选一个          |
| 可投资金融资产总额  | 金额    |  是 | 不含自住房；是否含养老金需明确                  |
| 不可承受损失资金   | 金额    |  是 | 未来三年刚性用途资金，不得计入高风险投资池            |
| 紧急备用资金     | 金额/月数 |  是 | 单独列示，不得与证券账户现金混淆                 |
| 投资目标       | 文本    |  是 | 写目标和期限，不写“赚钱”                    |
| 投资期限       | 枚举    |  是 | `<1年` / `1–3年` / `3–5年` / `5年以上` |
| 最大可接受总资产回撤 | 百分比   |  是 | 指会影响计划或行为的损失幅度                   |
| 允许资产类别     | 多选    |  是 | 股票、ETF、基金、债券、黄金、现金等              |
| 禁止资产类别     | 多选    |  是 | 未掌握产品、杠杆、裸期权、合约等                 |
| 策略分类       | 多选    |  是 | 核心配置 / 主动投资 / 交易 / 实验            |
| 各策略资金上限    | 百分比   |  是 | 每种策略占投资资产的最大比例                   |
| 基准组合或比较基准  | 文本    |  是 | 每类策略单独定义                         |
| 再平衡规则      | 文本    |  是 | 按日期或偏离阈值触发                       |
| 风险暂停规则     | 文本    |  是 | 达到哪些情况停止新增风险                     |
| 规则变更原则     | 文本    |  是 | 只能在月末或季度复盘后修改，紧急降风险除外            |

## 4.2 填写标准示例

### 投资目标

| 不合格写法 | 合格写法                            |
| ----- | ------------------------------- |
| 长期赚钱  | 用五年以上不用的资金实现长期购买力增长             |
| 稳健增值  | 在不影响未来三年现金需求的前提下，承担可接受波动以获得长期增值 |
| 提高收益  | 允许主动投资仓承担较高风险，但任何单一错误不能影响整体财务安全 |

### 最大可接受回撤

不能填写“尽量少亏”或“看情况”。

合格写法：

> 当投资资产自历史高点下跌达到 15% 时，停止新增主动风险并进行专项复核；达到 20% 时，重新评估风险资产比例、现金需求与策略有效性。该阈值不是自动清仓信号，而是治理动作触发线。

# 5. S01：账户主表

## 5.1 字段设计

| 字段         | 必填 | 填写标准                             |
| ---------- | -: | -------------------------------- |
| 账户 ID      |  是 | 例如 `ACC-CN-001`                  |
| 机构名称       |  是 | 券商、银行、基金平台完整名称                   |
| 账户类型       |  是 | 现金账户 / 保证金账户 / 基金账户 / 银行现金 / 养老金 |
| 所属市场       |  是 | A股 / 港股 / 美股 / 场外基金 / 其他         |
| 币种         |  是 | CNY / HKD / USD 等                |
| 是否允许融资或衍生品 |  是 | Yes / No                         |
| 是否纳入组合净值   |  是 | Yes / No，并注明原因                   |
| 初始录入日期     |  是 | 首次纳入系统的日期                        |
| 账户对账单存档位置  |  是 | 文件夹路径或链接                         |
| 数据更新方式     |  是 | 券商导出 / 手动 / API                  |
| 备注         |  否 | 税务、限售、特殊费用等                      |

## 5.2 规则

1. 所有持仓账户必须纳入系统，不允许只统计盈利账户。
2. 同一券商不同币种账户分开记录。
3. 保证金账户必须单独标记。
4. 不参与投资的生活现金不得混入证券现金。
5. 每月与券商对账单核对一次。

保持完整的交易、成本、分红、利息与账户记录，能够支持税务处理、费用核对以及发现账户差异。

## 6.2 流动性等级标准

| 等级 | 填写标准                              |
| -- | --------------------------------- |
| 高  | 正常市场条件下，可以在一个交易日内完成目标交易，且不会显著影响价格 |
| 中  | 需要分批交易，或成交价可能明显偏离预期               |
| 低  | 退出困难、存在封闭期、折价、停牌、赎回限制或显著冲击成本      |

## 6.3 风险主题标签标准

风险主题标签不是行业标签，而是判断多个仓位是否实际押注同一个因素。

例如：

| 标的       | 行业   | 风险主题           |
| -------- | ---- | -------------- |
| 半导体公司    | 科技   | AI资本开支、美元、成长估值 |
| 数据中心电力公司 | 公用事业 | AI资本开支、利率      |
| 科技指数 ETF | 多行业  | 美股成长估值、美元      |

因此，行业分散并不一定意味着风险分散。

| ------- | ---: | ----------------------------- |
| 交易 ID   |    是 | 唯一编号                          |
| 订单 ID   | 条件必填 | 有券商订单号时填写                     |
| 成交日期    |    是 | 实际成交日期                        |
| 成交时间    |   建议 | 主动交易建议填写                      |
| 账户 ID   |    是 | 关联 S01                        |
| 标的 ID   |    是 | 关联 S02                        |
| 策略 ID   |    是 | 核心配置 / 主动投资 / 交易 / 实验         |
| 论点 ID   | 条件必填 | 主动仓必须关联 S07                   |
| 决策单 ID  | 条件必填 | 主动买卖必须关联 S09                  |
| 操作类型    |    是 | 买入 / 卖出 / 申购 / 赎回 / 转入 / 转出   |

| 佣金      |    是 | 无费用填写 0                       |

| 基准货币成交额 |    是 | 换算后金额                         |
| 结算状态    |    是 | Pending / Settled / Corrected |
| 数据来源    |    是 | 券商成交记录或对账单                    |
| 更正关联 ID | 条件必填 | 若修正旧记录，指向原记录                  |

## 7.2 填写规则

1. 买入和卖出必须按实际成交记录，不按计划订单记录。
2. 撤单不进入交易流水，但可以进入决策记录。
3. 所有费用单独列示，不能简单混入成交价。
4. 转入资金、转出资金不属于收益。
5. 复权、拆股、分红、配股不作为普通买卖处理，而进入 S04。
6. 交易流水只记录事实，不填写“为什么买”。

| 资金加权收益率 MWR / XIRR | 评价个人资金实际体验 | 反映入金和出金时点        |

### 使用原则

* **评价策略能力**：使用 TWR。
* **评价自己投入资金最终获得的回报**：使用 MWR / XIRR。
* **不得只看账户总金额增长**，因为追加本金会伪装成投资收益。
* 组合收益应包含现金、分红、已实现盈亏、未实现盈亏与实际交易成本。CFA Institute 的 GIPS 计算方法强调，时间加权收益用于减少外部现金流对组合表现评价的影响，并要求方法持续一致。([GIPS][3])# 10.1 字段设计

## 14.2 风险预算指标

交易策略、事件策略和主动投资需要采用不同风险表达方式。

### A. 有明确止损的交易策略

```text
计划风险金额 = 数量 × |买入价 − 止损价| + 预计费用
计划风险比例 = 计划风险金额 / 组合净值
```

### B. 无机械止损的基本面投资

```text
情景风险金额 = 持仓市值 × 悲观情景跌幅
情景风险比例 = 情景风险金额 / 组合净值
```

### C. 核心配置策略

核心配置不使用单笔止损衡量风险，而使用：

* 目标资产配置偏离；
* 整体风险资产比例；
* 组合回撤；
* 现金需求覆盖情况；
* 再平衡是否触发。

宽基指数 ETF 与单一股票不能使用完全相同的集中度规则。

* **单一证券集中度限制**主要约束个股、单一债券、单一主题基金；
* **核心配置资产**主要通过资产类别比例、偏离幅度与再平衡规则控制。

| 修复动作           |    是 | 降仓、补录、暂停、更新规则等            |
| 是否需要系统修改       |    是 | Yes / No                  |
| 关闭日期           | 条件必填 | 问题解决后填写                   |

## 16.2 例外与违规的区别

| 类型   | 定义                     |
| ---- | ---------------------- |
| 事前例外 | 交易前发现不符合规则，但经过明确记录后仍执行 |
| 事后违规 | 交易完成后才发现或承认违反规则        |
| 数据错误 | 数字、分类、费用、现金流记录错误       |
| 流程遗漏 | 应写决策单、复核或事件跟踪但未完成      |

### 强制规则

* 同一规则三个月内出现两次事后违规，应进入季度系统整改。
* 例外交易不得因为赚钱而被视为“正确流程”。
* 违规是否严重，依据其风险后果与重复性，而不是最终盈亏。

# 17. 绩效与归因标准

## 17.1 必须计算的绩效指标

| 指标         |    频率 | 作用          |
| ---------- | ----: | ----------- |
| 月度净值变化金额   |    每月 | 看实际财富变化     |
| 月度 TWR     |    每月 | 看策略表现       |
| 年初至今 TWR   |    每月 | 看年度表现       |
| MWR / XIRR |   每季度 | 看自己的真实资金回报  |
| 分红与利息贡献    |    每月 | 区分收益来源      |
| 汇率贡献       |    每月 | 区分资产表现与币种影响 |
| 交易费用率      |    每月 | 识别换手成本      |
| 基准收益率      |    每月 | 判断主动行为是否有价值 |
| 超额收益       | 每月/季度 | 主动仓相对基准的表现  |

## 17.2 归因分类

每月的收益与亏损必须归入以下类别：

| 归因类别    | 示例             |
| ------- | -------------- |
| 资产配置贡献  | 股票相对现金的配置带来收益  |
| 标的选择贡献  | 某只股票优于其基准      |
| 仓位贡献    | 对判断正确的标的配置较高仓位 |
| 交易时点贡献  | 加减仓时间产生影响      |
| 汇率贡献    | 美元或港币变化        |
| 分红利息贡献  | 现金收入           |
| 费用拖累    | 交易、管理、融资成本     |
| 规则外行为贡献 | 未按计划交易产生的结果    |

## 17.3 基准选择标准

| 策略       | 基准标准             |
| -------- | ---------------- |
| 核心宽基配置   | 对应宽基指数或配置组合      |
| 行业主题投资   | 对应行业指数，不得只与现金比较  |
| 单一股票主动投资 | 同市场宽基 + 所属行业参考   |
| 短期交易     | 现金基准 + 策略自身历史记录  |
| 多币种投资    | 同币种收益与基准货币收益同时观察 |

### 禁止做法

* 主动投资亏损时改用更弱的基准；
* 盈利时只看绝对收益，不看承担的风险；
* 将入金导致的账户增长当作收益；
* 忽略分红、费用或汇率影响。

# 19. 每次交易前流程

## 19.1 交易前六步检查

| 步骤    | 问题         | 通过标准             |
| ----- | ---------- | ---------------- |
| 1. 分类 | 这是什么策略？    | 已明确属于核心、主动、交易或实验 |
| 2. 逻辑 | 为什么现在交易？   | 有事实、估值、规则或信号依据   |
| 3. 仓位 | 交易后占比多少？   | 不突破限制            |
| 4. 风险 | 错了会亏多少？    | 已填写悲观情景或计划损失     |
| 5. 失效 | 什么情况证明我错了？ | 有明确可观察条件         |
| 6. 行为 | 是否受情绪驱动？   | 情绪已记录，且不是唯一理由    |

## 19.2 一票否决项

出现以下任何一项，交易不得执行：

* 无法说明最大可能损失；
* 没有策略分类；
* 没有论点或没有入场规则；
* 交易后触及硬限制但未完成例外记录；
* 依据仅为传闻、社交媒体或他人仓位；
* 加仓理由仅为“成本降低”；
* 交易目的仅为快速挽回近期亏损。

## 22.2 季度判断标准

### 策略可以扩大额度，必须同时满足：

1. 至少有完整的交易与论点记录；
2. 收益来源能够归因于原计划，而非偶然重仓或市场普涨；
3. 没有重复性违规；
4. 风险和回撤在预设范围内；
5. 扩大额度后仍满足整体集中度限制。

### 策略应缩减或暂停，满足任一条件即可：

* 连续出现同类违规；
* 无法清晰解释收益来源；
* 风险远高于预期；
* 依赖高频决策但实际无法持续执行；
* 需要大量主观解释才能证明策略有效；
* 记录显示盈利主要来自一两次不可重复事件。

# 25. 第一版系统不应包含的内容

以下内容很容易制造复杂感，但在你尚未积累稳定记录前价值有限：

| 暂不优先建设           | 原因               |
| ---------------- | ---------------- |
| 实时盘口监控大屏         | 容易增加交易冲动，不提高决策质量 |
| 过多技术指标           | 未先验证策略逻辑，指标无意义   |
| 胜率排行榜            | 样本不足时误导严重        |
| 夏普率、复杂 VaR 等精细统计 | 数据历史不足时产生虚假精确感   |
| 自动推送大量新闻         | 信息噪声会超过决策价值      |
| 高复杂度自动交易         | 先建立人工可审计流程       |

# 28. 系统有效性的最终判定标准

三个月后，系统是否有效，不看你是否赚钱，而看以下十项：

| 判断问题              | 合格标准                 |
| ----------------- | -------------------- |
| 能否在十分钟内说清当前总风险？   | 能列出净值、回撤、最大仓位、最大主题暴露 |
| 每个主动仓是否有明确存在理由？   | 100% 有论点卡            |
| 是否能识别哪些信息真正改变了决策？ | 重要交易均关联信息来源          |
| 是否能区分本金变动与投资收益？   | 现金流和收益分离             |
| 是否能区分策略表现与个人资金体验？ | 同时有 TWR 与 MWR/XIRR   |
| 是否存在无计划交易？        | 数量下降且有记录             |
| 是否出现规则被盈利掩盖？      | 所有例外都记录，不因盈利豁免       |
| 是否能够指出自己最常犯的错误？   | 有错误分类和整改动作           |
| 是否能够说明主动投资是否优于基准？ | 有同类基准比较              |
| 系统是否减少而不是增加冲动交易？  | 交易频率与违规次数受控          |

---

## 最关键的落地原则

不要把系统建设成“证明自己有道理”的资料库，而要建设成一个不断回答以下问题的机制：

> **在交易发生之前，我是否知道自己为什么做、错了会损失多少、什么情况必须停止；在交易发生之后，我是否能够诚实区分能力、风险、运气与违规。**

这才是从小仓位直觉投资，过渡到可管理的大仓位决策体系的关键。
