# 迭代 P0 多轮澄清

## 第 1 轮：策略是否立刻替换历史 `strategy_type`

判断：现有交易、论点、复核事件已有 `strategy_type`，直接替换会扩大迁移风险。

收束：P0 不删除旧字段，新增策略实体和策略版本。旧字段继续兼容，新能力优先引用 `strategy_id` / `strategy_version_id`。

## 第 2 轮：AI 运行是否必须调用外部模型

判断：本地测试和散户工作台不能依赖外部 API 才可用。已有模型调用能力应保留，但迭代入口首先要保证可审查工作流能落地。

收束：P0-P4 的新 workflow 先以本地数据生成结构化阶段输出，并持久化；后续可把阶段输出替换为外部模型生成。

## 第 3 轮：三入口是否要拆成三个页面

判断：研究工作台当前已采用 Tab 组合。过早拆页面会增加导航复杂度。

收束：P0 先在 AI 研究面板里提供三入口选择，策略和复盘结构化记录作为研究工作台 Tab 展示。

## 第 4 轮：是否自动生成交易决策

判断：面向散户应保留人工确认，AI 不应自动写交易流水。

收束：P0 只生成可审查建议和运行记录；后续生成交易决策草案也必须由用户确认。
