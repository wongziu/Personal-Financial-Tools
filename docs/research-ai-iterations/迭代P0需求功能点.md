# 迭代 P0 需求功能点

## 目标

先补产品语义，不急着补模型能力。研究工作台必须能清楚表达三种 AI 触发入口：策略运行、标的诊断、复盘会话；并持久化每次 AI/Agent 运行的输入、阶段和结论。

## 功能点

1. 策略成为可引用实体，而不是只在交易、论点中使用 `strategy_type` 枚举。
2. AI 运行结果持久化，包括运行类型、关联标的、关联策略、问题、模型、状态、最终摘要。
3. AI 阶段结果持久化，包括阶段顺序、阶段 ID、标题、状态、输入摘要、输出、耗时。
4. 研究工作台新增三入口语义：策略运行、标的诊断、复盘会话。
5. 旧的单次 AI 分析和五段 Agent 工作流继续保留，避免破坏现有使用路径。

## 验收标准

- 数据库存在策略、策略版本、AI 运行、AI 阶段表。
- 研究工作台 UI 能看到策略、标的、复盘三类迭代入口。
- 新入口执行后能返回并展示持久化运行 ID 和阶段结果。
- 单元测试验证运行结果写入数据库。
