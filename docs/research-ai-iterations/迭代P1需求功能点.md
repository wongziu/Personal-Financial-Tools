# 迭代 P1 需求功能点

## 目标

交付策略实验室 MVP。用户可以定义散户策略，运行策略，并获得候选卡片。候选卡片不直接要求交易，而是给出观察、补证据、建论点或生成决策草案的下一步。

## 功能点

1. 策略版本记录适用账户、市场、候选规则、证据要求、仓位上限、复盘频率。
2. 策略运行读取本地标的、信息来源、论点、价格和交易决策。
3. 策略候选卡片保存候选标的、排名、适配分、命中规则、缺失证据、风险标记、下一步动作。
4. 策略运行结果保存数据覆盖情况和最终摘要。

## 验收标准

- demo 数据中存在策略和策略版本。
- 调用策略运行后生成至少一个候选卡片。
- 候选卡片能在研究工作台中查看。
- 测试覆盖候选卡片生成规则。
